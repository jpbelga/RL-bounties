RL Model Bounties in Bug Buster

Team: Pedro Sarmento Lins, João Pedro Belga, Leonardo Santos de Camargo

Client: Cartesi

Date: 10/08/2024

Introduction

Bug Buster: A trustless bug bounty platform powered by Cartesi Rollups.

Initial Goal: Add reinforcement learning (RL) model bounties to the existing system.

Why RL Models?: Expands Bug Buster's use case beyond bug hunting—introduces a marketplace for AI model competitions in a trustless environment.

Project Overview

Traditional Bug Bounties:

  • Focused on rewarding hackers for discovering software vulnerabilities.
  • Limited to bug hunting and exploit submissions.
  • Trustless payment.

RL Model Bounties:

  • Focused on rewarding hackers for training the best RL model in an environment of choice.
  • Tournament format.
  • Trustless payment.

Project Overview

New Bounty Type: RLModelBounty.

How It Works:

  • Sponsors create RL model bounties with ERC-20 rewards.
  • Hackers submit models (ONNX format).
  • Models are evaluated based on performance (higher score wins).

Result: Expands Bug Buster to host AI challenges and model competitions.

Architecture Before vs. After

Before:

  • Supported traditional bug bounties only.
  • Focus on discovering vulnerabilities.

After:

  • Supports both RL model and bug bounties.
  • Model submission mechanism.
  • Scoring and ranking system for RL models.

Frontend Changes

  • Explore: Added filters for bug bounties and RL model bounties.
  • Exploit/[id]: RL bounties: Option to download models. Local environment to test models before submission.
  • Model Interface: New fields for BountyType and Attempts.

Dapp Changes

  • AppBounty: New BountyType: RLModelBounty or BugBounty. Attempts stores hacker, inputIndex, and score for RL model submissions.
  • InputKinds: Expanded to support both RLModelBounty and BugBounty.
  • Automatic Scoring: Submissions ranked by score after deadline (highest score wins).

Bounties Update

  • New RLModelBounty: environment.py: Helper code for local development. test.py: Executes model and outputs score.
  • BugBounty: Remains the same as before (program + test scripts in tar file).
  • Tar File Changes: RLModelBounty tar now includes ONNX model execution logic.

OS Environment Changes

  • New Environment: Moved from Ubuntu Noble to slim-jammy.
  • Installed cp310 ONNX Runtime wheel.
  • Increased memory from 128Mi to 1024Mi for handling ONNX models.

Final Results

New Bounty Type: Successfully added RL model bounties to Bug Buster.

Key Outcomes:

  • Opened up a marketplace for AI model challenges.
  • Expanded use case for Bug Buster beyond traditional bug hunting.

Challenges in Development

  • Rebuild Times: Slow rebuilds of the Cartesi image hindered rapid iteration.
  • Debugging Tools: Lack of tools for inspecting variables in real-time or debugging.
  • CLI Issues: Required building the image twice for changes to take effect.
  • Authentication for Rollups: No equivalent for an authentication mechanism within rollups.

Cartesi Development Environment

  • Positive Aspects: Deterministic Linux environment made blockchain integration seamless.
  • Flexibility: Traditional programming tools within blockchain context.
  • Learning Curve: Minimal prior experience with blockchain development; quick adaptation to Cartesi’s infrastructure.

Next Steps

  • Future Improvements: Optimize rebuild times and streamline CLI usage. Add more introspection tools for debugging during development.
  • New Opportunities: Explore expanding to other types of AI challenges. Further develop the model marketplace for broader applications.

Conclusion

Successful Outcome: RL Model Bounties integrated into Bug Buster.

Enhanced Platform: Bug Buster now supports both bug bounties and AI model competitions.

Great Experience: Despite challenges, our team successfully navigated the Cartesi environment and expanded Bug Buster's capabilities.
